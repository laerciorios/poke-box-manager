All OpenSpec artifacts must be written in English — proposals, design docs, specs, and tasks files.

**Why:** The project's existing OpenSpec changes (drag-and-drop, pokemon-detail-card, remove-game-availability) are all in English. The user corrected a change written in Portuguese and asked it to be translated.

**How to apply:** Always write `proposal.md`, `design.md`, `specs/**/*.md`, and `tasks.md` in English, regardless of the conversation language. Conversation with the user can remain in Portuguese.
